```

4. AI 透明可审计 (Transparent AI)
5. 逻辑处处复用 (Logic Everywhere)
```

## 🧩 核心功能模块详解

### 1️⃣ Notebook - 响应式分析本

** Querio 核心灵感**: 像电子表格一样响应式，像 Jupyter 一样灵活

#### 核心特性

```python
# ===== Notebook 示例：销售分析 =====

# Cell 1: 数据加载 (响应式)
orders = sql("SELECT * FROM orders WHERE created_at >= '2026-01-01'")
customers = sql("SELECT * FROM customers")

# Cell 2: 数据转换 (自动依赖追踪)
active_customers = customers.filter(last_order_date >= '2026-02-01')
orders_by_month = orders.group_by('month').agg(total='SUM(amount)')

# Cell 3: 指标计算 (复用定义)
gmv = sql("SELECT SUM(amount) as gmv FROM orders")
mom_growth = (gmv.current - gmv.last_month) / gmv.last_month

# Cell 4: AI 分析 (透明可审计)
# @ai: 分析本月 GMV 下降的原因
insight = ai.analyze(
    data=orders,
    question="为什么本月 GMV 下降了？",
    context=[active_customers, orders_by_month]
)

# Cell 5: 可视化
chart(orders_by_month, type='line', title='月度销售趋势')
chart(insight.contributing_factors, type='bar')

# Cell 6: 发布为看板
board = publish([gmv, mom_growth, chart1, chart2], 
                title='销售监控看板',
                refresh='daily')
```

#### 关键技术点

| 特性 | 说明 | 实现方式 |
|------|------|----------|
| 响应式计算 | 单元格自动重新计算 | 依赖图 + 变更传播 |
| SQL + Python | 混合编程环境 | 统一 AST 解析 |
| AI 透明性 | 每个 AI 查询都是显式代码 | Code-first AI |
| 版本控制 | Notebook 是.py 文件 | Git 友好 |
| 逻辑复用 | 可导入其他 Notebook | import 机制 |

```python
# context/metrics/gmv.py

from bizlens import Metric, sql

gmv = Metric(
    name="GMV",
    description="成交总额",
    formula=sql("SUM(orders.amount)"),
    tables=["orders"],
    tags=["销售", "核心指标"],
    owner="data-team"
)

gmv_daily = Metric(
    name="日 GMV",
    description="每日成交总额",
    formula=sql("DATE(created_at), SUM(amount)"),
    grain="day",
    based_on="gmv"
)

gmv_mom = Metric(
    name="GMV 环比增长率",
    description="GMV 月度环比增长",
    formula="(本月 GMV - 上月 GMV) / 上月 GMV",
    format="percentage"
)
```

```python
# context/skills/churn_analysis.py

from bizlens import Skill

churn_analysis = Skill(
    name="客户流失分析",
    description="分析客户流失原因并生成挽回建议",
    
    triggers=[
        "流失分析",

        "orders",
        "customers", 
        "order_items"

    output_schema={
        "churn_rate": "float",
        "churned_customers": "DataFrame",
        "top_reasons": "List[str]",
        "recommendations": "List[str]"
    }
```

### 3️⃣ Explore - 智能探索器

**自然语言查询 + AI Agent 协作**

#### 查询流程

```
用户提问 → Query Agent 解析 → 语义层映射 → SQL 生成 → 结果展示 → AI 解读
```

#### 示例对话

```
用户: "帮我看看上个月各品类的销售情况"

Query Agent:
  1. 识别意图：分类查询 + 时间筛选
  2. 映射指标：销售 → GMV
  3. 映射维度：品类 → category
  4. 时间解析：上个月 → 2026-02-01 ~ 2026-02-29
  5. 生成 SQL:
     SELECT category, SUM(amount) as gmv 
     FROM orders 
     WHERE created_at BETWEEN '2026-02-01' AND '2026-02-29'
     GROUP BY category
     ORDER BY gmv DESC

结果展示:
┌──────────────┬──────────────┬─────────────┐
│ 品类         │ GMV          │ 占比        │
├──────────────┼──────────────┼─────────────┤
│ 电子产品     │ ¥1,250,000   │ 35%         │
│ 服装         │ ¥890,000     │ 25%         │
│ 家居         │ ¥720,000     │ 20%         │
│ 美妆         │ ¥540,000     │ 15%         │
│ 其他         │ ¥180,000     │ 5%          │
└──────────────┴──────────────┴─────────────┘

AI 洞察:
- 电子产品占比最高 (35%)，主要由手机贡献
- 美妆品类环比增长最快 (+28%)
- 家居品类连续 3 个月下滑，需关注
```

### 核心组件技术选型

#### 1. Notebook 引擎

```yaml
方案：自研响应式引擎 + Pyodide

理由:
  - Pyodide: 浏览器端 Python 执行，安全沙箱
  - 响应式：基于有向无环图 (DAG) 的依赖追踪
  - 混合计算：SQL → DuckDB, Python → Pyodide

核心接口:
  - sql(query: str) → DataFrame
  - ai.analyze(data, question, context) → Insight
  - chart(data, type, options) → Visualization
  - publish(cells, title, refresh) → Board
```

#### 2. 语义层

```yaml
方案：Go 实现 + PostgreSQL 存储

数据结构:
  - metrics: 指标定义表
  - dimensions: 维度定义表
  - relationships: 关系定义表
  - business_rules: 业务规则表

查询流程:
  1. 解析自然语言
  2. 语义层查找匹配指标/维度
  3. 生成 SQL
  4. 质量检查 (血缘、性能)
  5. 执行 + 缓存
```

#### 3. AI Agent

```yaml
方案：多 Agent 协作 + Function Calling

Agent 编排:
  - Query Agent: 意图识别 → SQL 生成
  - Semantic Agent: 语义映射 → 业务规则
  - Insight Agent: 异常检测 → 根因分析
  - Quality Agent: SQL 验证 → 性能优化

AI 模型:
  - 支持多模型：OpenAI/Claude/ 通义/文心
  - 本地缓存：Prompt + Response 缓存
  - 可审计：所有 AI 决策记录日志
```
