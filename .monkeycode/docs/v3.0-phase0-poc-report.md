```

执行顺序测试:
```

### 限制与应对

| 限制 | 影响 | 应对措施 |
|------|------|----------|
| 初始加载慢 (3-5s) | 用户体验 | 预加载 + 缓存 + Loading 动画 |
| 内存占用高 (100MB) | 浏览器压力 | 按需加载 + 内存管理 |
| 部分库不支持 | 功能限制 | 白名单机制 + 降级方案 |
| 文件系统隔离 | 数据持久化 | IndexedDB + 虚拟文件系统 |

### 性能优化建议

```yaml
加载优化:
  - Service Worker 缓存 Pyodide 核心文件
  - Web Worker 后台加载
  - 懒加载 Python 库

执行优化:
  - 代码分割
  - 增量计算
  - 结果缓存

内存优化:
  - 及时释放不用的变量
  - 限制 DataFrame 大小
  - 垃圾回收触发
```

### 结论

✅ **Pyodide 技术完全可行**

- 核心功能满足 Notebook 需求
- 性能可接受（简单计算 <100ms）
- 需要优化加载速度和内存管理

## 3️⃣ 语义层数据模型设计

### 实现内容

**文件位置**: `backend/pocs/semantic-layer-schema.sql`

**核心表**:
- 指标定义表 (metrics)
- 维度定义表 (dimensions)
- 业务关系表 (relationships)
- Notebook 表 (notebooks)
- 看板表 (boards)
- AI 技能表 (ai_skills)

### 设计亮点

```sql
-- ✅ 指标定义支持 Git 版本控制
CREATE TABLE metrics (
    -- ...
    git_path VARCHAR(500),      -- Git 路径
    git_commit VARCHAR(40),     -- Commit hash
    -- ...
);

-- ✅ 维度支持层次结构
CREATE TABLE dimensions (
    -- ...
    hierarchy JSONB,            -- {"parent": "province", "children": ["city"]}
    sample_values TEXT[],       -- 示例值
    -- ...
);

-- ✅ Notebook 支持响应式依赖
CREATE TABLE notebooks (
    -- ...
    cells JSONB,                -- [{id, type, code, output, deps}]
    dag JSONB,                  -- 依赖图
    -- ...
);

-- ✅ AI 技能可复用
CREATE TABLE ai_skills (
    -- ...
    triggers TEXT[],            -- 触发关键词
    instructions TEXT,          -- Prompt template
    output_schema JSONB,        -- 输出定义
    -- ...
);
```

### 索引优化

```sql
-- ✅ 组织隔离索引
CREATE INDEX idx_metrics_org ON metrics(organization_id);

-- ✅ 常用查询优化
CREATE INDEX idx_metrics_name ON metrics(organization_id, name);

-- ✅ 标签搜索
CREATE INDEX idx_metrics_tags ON metrics USING GIN(tags);

-- ✅ 核心指标过滤
CREATE INDEX idx_metrics_core ON metrics(is_core) WHERE is_core = true;
```

### 数据完整性

```sql
-- ✅ 外键约束
REFERENCES organizations(id)
REFERENCES data_sources(id)
REFERENCES users(id)

-- ✅ 唯一约束
UNIQUE(organization_id, name)
UNIQUE(notebook_id, version)

-- ✅ 软删除
deleted_at TIMESTAMPTZ
```

### 视图和函数

```sql
-- ✅ 核心指标视图
CREATE VIEW core_metrics AS
SELECT m.*, 
       COUNT(DISTINCT n.id) as used_in_notebooks,
       COUNT(DISTINCT b.id) as used_in_boards
FROM metrics m
-- ...
WHERE m.is_core = true;

-- ✅ 热门查询视图
CREATE VIEW popular_queries AS
SELECT question, COUNT(*), AVG(execution_time)
FROM query_history
WHERE created_at > NOW() - INTERVAL '30 days'
GROUP BY question;
```

- Git 版本控制友好
- 组织隔离和数据安全
