# 后台管理功能需求文档

> BizLens AI 数据分析平台 - 后台管理系统

## 1. 概述与目标

### 1.1 背景

当前 BizLens 平台的功能分散在多处：
- 数据源管理：`/settings/data-sources`（数据库连接 + 文件上传 + 指标管理）
- AI 配置：`/settings/ai`
- 企业档案：`/settings/profile`

用户希望有一个集中的**后台管理**入口，将"上传文件"和"配置数据库"等系统管理功能统一收纳，同时扩展更多管理能力（如文件管理、存储配置、多租户管理等）。

### 1.2 目标

构建一个集中的后台管理系统 `/admin`，统一纳管：
1. **数据资产管理**：文件管理中心 + 数据库连接管理 + 存储配置
2. **多租户管理**：租户列表 + 用户管理（面向 SaaS 运营商）
3. **系统概览**：仪表盘展示系统状态和关键指标

### 1.3 UI 风格决策

**决策：延续现有风格，独立路由 `/admin`**

- 复用现有 `globals.css` 设计系统（深色主题 + 毛玻璃 + Tailwind）
- 复用现有 `AppHeader` 组件，统一顶部导航
- 独立 `AdminLayout` 提供侧边导航和后台专属样式
- 不建设独立 UI 系统，保证视觉一致性

## 2. 功能需求

### 2.1 后台管理入口

**2.1.1 入口显示条件**
- 仅 `role === 'admin' || role === 'owner'` 的用户可见
- 在 AppHeader 导航栏增加"管理"菜单（下拉或独立入口）

**2.1.2 导航结构**
```
BizLens
├── AI 对话          /chat
├── 洞察            /insights
├── 观测中心        /dashboards
├── 报表            /reports
├── 告警            /alerts
├── 设置            /settings/*
└── 管理 [admin]    /admin/*
    ├── 仪表盘        /admin
    ├── 数据资产      /admin/assets
    │   ├── 文件管理  /admin/assets/files
    │   ├── 数据库    /admin/assets/databases
    │   └── 存储配置  /admin/assets/storage
    ├── 租户管理      /admin/tenants
    │   ├── 租户列表  /admin/tenants
    │   └── 用户管理  /admin/tenants/users
    └── 系统配置      /admin/config
```

### 2.2 仪表盘（/admin）

**功能：**
- 展示系统总览卡片：总租户数、总用户数、总文件数、总数据库连接数
- 最近上传文件列表（5条）
- 存储使用情况概览（饼图或进度条）
- 活跃租户趋势（7天内有操作的租户）
- 系统健康状态指示

**数据来源：**
- 调用后端 `/api/admin/stats` 获取汇总数据
- 仅 admin/owner 可访问

### 2.3 数据资产管理

#### 2.3.1 文件管理（/admin/assets/files）

**功能：**
- 列出所有租户的上传文件（支持筛选租户、文件类型、上传时间）
- 文件列表字段：文件名、所属租户、文件大小、格式、上传时间、状态、操作
- 支持按文件名、租户名搜索
- 支持分页（每页 20 条）
- 单个文件操作：预览（图片/PDF）、下载、删除
- 批量选择并删除
- 统计：当前文件总数、总存储大小、各格式文件数量分布

**交互：**
- 表格顶部筛选栏：租户选择（admin 可选任意租户）、文件类型下拉、时间范围
- 表格行悬停显示操作按钮
- 删除操作需二次确认弹窗
- 大文件（>10MB）显示进度条

**复用/扩展：**
- 复用现有 `FileUploadTab` 的上传组件逻辑
- 复用现有 `/api/datasets` 和 `/api/datasets/upload/file` API
- 新增 admin 专属列表 API `GET /api/admin/datasets`，支持租户过滤

#### 2.3.2 数据库连接管理（/admin/assets/databases）

**功能：**
- 列出所有租户的数据库连接配置（支持筛选租户、数据库类型、连接状态）
- 数据库列表字段：名称、所属租户、类型（MySQL/PostgreSQL/SQLite/API）、连接状态、最近同步时间、操作
- 支持按名称、租户名搜索
- 支持分页
- 单个连接操作：测试连接、查看详情、删除
- 连接诊断详情弹窗（复用 `ConnectionDiagnosis` 结果）

**交互：**
- 表格顶部筛选栏：租户选择、数据库类型下拉、连接状态筛选
- "测试连接"按钮显示加载状态和结果图标
- 详情弹窗展示完整连接信息（密码脱敏显示）
- 删除需二次确认

**复用/扩展：**
- 复用现有 `DatabaseConnectionTab` 的连接表单和测试逻辑
- 复用现有 `/api/tenants/{id}/data-sources` API
- 新增 admin 列表 API `GET /api/admin/data-sources`，支持租户过滤
- 新增 `POST /api/admin/data-sources/{id}/test` 连接测试 API

#### 2.3.3 存储配置（/admin/assets/storage）

**功能：**
- 查看当前存储配置（本地存储 / S3 兼容对象存储）
- 配置存储后端类型（本地 / S3 / MinIO）
- S3 配置项：Endpoint、AccessKey、SecretKey、Bucket、Region
- 本地存储路径配置
- 存储使用统计：各租户占用空间排行
- 一键清理：清理过期文件（>30天未访问）

**交互：**
- 表单式配置，字段根据存储类型动态显示
- 保存后端类型切换需确认
- 清理操作需二次确认，显示待清理文件数量

### 2.4 租户管理（/admin/tenants）

#### 2.4.1 租户列表（/admin/tenants）

**功能：**
- 列出所有租户
- 租户列表字段：租户名称、Plan（free/pro/enterprise）、用户数、数据源数、文件数、创建时间、状态、操作
- 支持按名称搜索
- 支持分页
- 租户操作：创建租户、编辑租户信息、切换 Plan、暂停/启用租户、删除租户

**交互：**
- 表格行支持展开详情
- 创建/编辑租户弹窗表单：名称、Plan 选择、备注
- 删除租户需二次确认（警告：删除租户将清除所有数据）

#### 2.4.2 用户管理（/admin/tenants/users）

**功能：**
- 按租户筛选或查看全局用户
- 用户列表字段：用户名、邮箱、角色（owner/admin/member）、所属租户、最后登录时间、状态、操作
- 支持搜索（邮箱、用户名）
- 用户操作：创建用户、编辑用户角色、重置密码、禁用/启用用户、删除用户

**交互：**
- 左侧可选租户筛选（"全部"或具体租户）
- 表格顶部搜索框
- 创建用户弹窗：用户名、邮箱、初始密码、角色、所属租户
- 角色切换即时生效

### 2.5 系统配置（/admin/config）

**功能：**
- AI 模型默认配置（fallback 配置）
- 系统通知配置（邮件通知、SMS 通知）
- 告警全局配置（默认通知渠道、告警冷却时间）
- 数据保留策略（文件保留天数、告警日志保留天数）
- API 限流配置

**交互：**
- 分区块的表单配置，每块有标题和说明
- 保存按钮在各区块底部，单独保存
- 敏感配置（API Key）需点击"显示"后可见

## 3. 非功能需求

### 3.1 权限控制

- 后台管理路由需 `admin` 或 `owner` 角色
- 未授权用户访问 `/admin/*` 时重定向到 `/chat`
- admin 用户可管理所有租户数据
- owner 用户仅可管理其所属租户数据（等同于普通用户 + 后台只读视图）

### 3.2 多租户隔离

- 所有 admin API 默认不携带租户过滤时返回全部数据
- 所有 admin API 需管理员认证
- 数据库操作严格按 tenant_id 隔离

### 3.3 性能要求

- 列表 API 支持分页（默认 20 条/页）
- 文件列表加载时间 < 2s（不含大文件预览）
- 数据库连接测试超时 10s

### 3.4 安全要求

- 密码字段在前端不显示，传输时加密
- 敏感配置（API Key、数据库密码）脱敏展示
- 所有 admin API 需 JWT 认证
- 关键操作（删除租户、清空存储）需二次确认

## 6. 验收标准

1. admin/owner 用户可在 AppHeader 中看到"管理"入口
2. `/admin` 路由在非 admin 用户访问时重定向
3. 仪表盘正确显示系统汇总数据
4. 文件管理可列出所有租户的文件，支持筛选、搜索、分页
5. 数据库管理可列出所有租户的数据库连接，支持测试连接
6. 租户管理可进行 CRUD 操作
7. 用户管理可进行 CRUD 操作
8. 所有操作符合现有 UI 风格，无突兀感
9. 关键操作有确认提示
10. 敏感信息脱敏展示
