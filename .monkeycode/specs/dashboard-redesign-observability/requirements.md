# 数据大屏重构需求文档 -- 业务观测中心

> 版本: v1.0
> 日期: 2026-03-26
> 状态: Draft

---

## 1. 背景与动机

### 1.1 现状问题

当前数据大屏模块（`/dashboards`）是一个**静态模板展示系统**：
- 提供 9 个行业预置模板（销售分析、电商运营、SaaS 指标等）
- 用户选择模板 -> 预览示例数据 -> 保存为"我的大屏"
- AI 对话可生成 `dashboard_config` JSON -> 自动创建大屏实例
- 所有图表数据为**静态示例数据**（硬编码在模板 JSON 中），未接入真实数据源

### 1.2 与产品战略的差距

根据 `product-direction-2026Q2.md` 的"业务雷达站"定位：

| 维度 | 当前大屏 | 业务雷达站要求 |
|------|---------|--------------|
| 数据来源 | 静态示例数据 | 真实数据源实时查询 |
| 交互模式 | 用户手动选模板浏览 | 系统主动推送异常 + 用户按需下钻 |
| 核心价值 | "展示好看的图表" | "监控业务健康、发现并解释异常" |
| AI 角色 | 生成 JSON 模板配置 | 基线学习 + 异常检测 + 根因分析 + 洞察生成 |
| 数据资产 | 无积累（一次性模板） | 基线数据 + 异常历史 + 指标定义持续积累 |
| 用户粘性 | 低（看完就走） | 高（每日摘要推送、异常告警、健康评分） |

### 1.3 重构目标

将数据大屏从"静态模板画廊"重构为"业务健康观测中心"，使其成为用户每天打开 BizLens 的第一个页面，实现：

1. **主动监控**：连接数据源后自动建立基线、检测异常、推送告警
2. **健康评分**：用一个数字概括业务整体健康状况
3. **智能洞察**：AI 主动发现趋势、异常、机会，而非等用户提问
4. **根因下钻**：点击异常指标可自动展开多维度根因分析
5. **保留模板能力**：模板系统作为"自定义看板"的子功能保留，降低展示权重

## 2. 功能需求

### FR-1: 业务健康总览页（替代现有大屏首页）

**优先级**: P0

当用户导航到 `/dashboards` 时，系统应展示业务健康总览页，而非模板画廊。

- **FR-1.1**: 系统应在页面顶部展示一个**业务健康评分**（0-100），包含评分数值、评分趋势（较昨日升/降）、评分等级标签（优秀/良好/注意/警告/危险）
- **FR-1.2**: 系统应展示**核心指标速览卡片**（KPI 区域），每张卡片包含：指标名称、当前值、同比/环比变化率、迷你趋势图（Sparkline，近 7 天）、异常状态标识（正常/预警/异常）
- **FR-1.3**: 核心指标应基于已接入数据源的语义层指标自动选择，优先展示标记为"核心指标"的指标，若未配置语义层则展示引导提示
- **FR-1.4**: 系统应展示**异常事件流**（Anomaly Feed），按时间倒序展示最近检测到的异常事件，每条包含：异常指标名称、偏离程度、严重等级（critical/warning/info）、检测时间、简要说明
- **FR-1.5**: 用户点击异常事件应可展开查看**根因分析摘要**，包括关联指标变化、维度下钻结果、AI 生成的假设和建议
- **FR-1.6**: 系统应展示**AI 洞察卡片**（Insight Cards），由 AI 主动生成的业务洞察，如"连续 3 天增长趋势""某渠道 ROI 创新高"等正面/负面/中性洞察
- **FR-1.7**: 页面应支持时间范围切换（今天/本周/本月/自定义），所有数据组件联动刷新

### FR-2: 每日业务摘要

**优先级**: P0

- **FR-2.1**: 系统应每日定时（可配置，默认 09:00）生成业务摘要
- **FR-2.2**: 每日摘要应包含：健康评分、核心指标速览（值+同比+环比）、需关注事项（异常列表）、正面趋势、预测展望
- **FR-2.3**: 每日摘要应可通过已配置的 IM 渠道（飞书/钉钉/企微）推送
- **FR-2.4**: 用户在观测中心页面应可查看历史每日摘要列表

### FR-3: 异常检测与告警

**优先级**: P0

- **FR-3.1**: 系统应对所有已注册的语义层指标自动建立统计基线（移动平均法），学习窗口默认 30 天
- **FR-3.2**: 系统应按配置的粒度（小时/天）定时执行异常检测，将实际值与基线对比
- **FR-3.3**: 异常检测结果应包含：严重等级、置信度、偏离方向和幅度、上下文说明
- **FR-3.4**: 系统应实现降噪策略：同一指标同方向 4 小时内最多推送 1 次、持续异常但无恶化时自动降级严重度
- **FR-3.5**: 检测到 critical 级异常时，系统应通过 IM 渠道主动推送告警
- **FR-3.6**: 用户应可在观测中心对异常事件进行反馈：已确认（acknowledge）、已解决（resolve）、误报（false positive）

### FR-4: 根因分析

**优先级**: P1

- **FR-4.1**: 当异常事件被确认后，系统应自动触发根因分析
- **FR-4.2**: 根因分析应执行维度下钻（遍历语义层已定义的维度，找到贡献度最高的维度值组合）
- **FR-4.3**: 根因分析结果应包含：假设列表（含置信度和贡献度）、数据证据、验证建议
- **FR-4.4**: 根因分析结果应在观测中心的异常事件详情中展示，支持用户继续追问（跳转 AI 对话，预填上下文）

### FR-5: 自定义看板（保留并简化现有模板功能）

**优先级**: P1

- **FR-5.1**: 用户应可创建自定义看板，选择要展示的指标和图表类型
- **FR-5.2**: 系统应保留现有模板作为"看板模板"，用户可基于模板快速创建看板
- **FR-5.3**: 自定义看板应支持绑定真实数据源，图表数据从数据源实时查询
- **FR-5.4**: AI 对话中生成的大屏配置应创建为自定义看板（而非旧版静态大屏实例）
- **FR-5.5**: 自定义看板功能入口从页面主体移至二级入口（如"我的看板" Tab 或设置区域）

### FR-6: 趋势预测（可视化展示）

**优先级**: P2

- **FR-6.1**: 观测中心的趋势图表应支持叠加显示未来 N 天的预测值（虚线）和置信区间（阴影带）
- **FR-6.2**: 预测模型应基于指标的历史基线数据自动生成，无需用户配置
- **FR-6.3**: 预测结果应包含置信度说明和关键驱动因素

## 3. 非功能需求

### NFR-1: 性能

- **NFR-1.1**: 观测中心首页加载时间不超过 3 秒（含核心指标查询）
- **NFR-1.2**: 异常检测定时任务单次执行时间不超过 30 秒（100 个指标以内）
- **NFR-1.3**: 根因分析单次执行时间不超过 60 秒

### NFR-2: 可用性

- **NFR-2.1**: 未配置数据源时，观测中心应展示引导界面而非空白页，引导用户完成数据源连接和指标配置
- **NFR-2.2**: 语义层指标不足时，系统应展示可用的指标数量和"配置更多指标"的入口

### NFR-3: 向后兼容

- **NFR-3.1**: 已创建的旧版大屏实例应在"我的看板"中仍可查看和编辑
- **NFR-3.2**: 旧版 `/dashboard` 路由重定向逻辑保持不变
- **NFR-3.3**: AI 对话中的 `dashboard_config` 生成和解析逻辑保持兼容

### NFR-4: 安全性

- **NFR-4.1**: 所有新增 API 应复用现有 JWT 认证中间件
- **NFR-4.2**: 异常事件、基线数据、摘要数据按 tenant_id 隔离

## 4. 页面信息架构（重构后）

```
/dashboards (观测中心 -- 默认首页)
  |
  +-- 业务健康评分 (顶部)
  +-- 核心指标速览 (KPI 卡片网格)
  +-- 异常事件流 (时间线)
  |     +-- 事件详情 -> 根因分析摘要
  |     +-- "在 AI 对话中追问" 按钮
  +-- AI 洞察卡片
  +-- 趋势图表区 (含预测)
  +-- 每日摘要历史 (折叠区域)
  |
  +-- [Tab/按钮] 我的看板
        +-- 已创建的自定义看板列表
        +-- 创建新看板 (模板选择 / 空白画布 / AI 生成)
```

## 5. 与现有模块的关系

| 现有模块 | 关系 | 说明 |
|---------|------|------|
| AlertEvent + AlertTriggerLog | **扩展** | 新增 baseline 类型的告警，复用现有告警基础设施 |
| IM 适配器（飞书/钉钉/企微） | **复用** | 异常推送和每日摘要通过已有 IM 通道发送 |
| 语义层（Metric/Dimension） | **依赖** | 观测中心的核心指标和异常检测基于语义层指标定义 |
| AI 对话 (ChatPanel) | **双向集成** | 异常事件可跳转对话追问；对话中可生成看板 |
| Schema 感知系统 | **间接依赖** | 语义层不足时退化为 schema 感知模式 |
| DashboardTemplate/Instance | **保留降级** | 作为"自定义看板"子功能保留，不再是大屏首页 |

## 6. 验收标准

### AC-1: 观测中心首页
- 接入数据源且配置了至少 3 个语义层指标后，打开 `/dashboards` 应看到健康评分、KPI 卡片、异常事件流
- 无数据源时应看到引导页面

### AC-2: 异常检测
- 手动插入偏离基线的数据后，系统应在下一次检测周期内生成 anomaly_event 记录
- critical 级异常应通过 IM 推送（如已配置 IM 渠道）

### AC-3: 每日摘要
- 系统应在配置时间生成当日摘要，内容包含健康评分和核心指标
- IM 渠道推送的摘要格式应清晰可读

### AC-4: 向后兼容
- 旧版大屏实例在"我的看板" Tab 中仍可正常渲染
- AI 对话中 `dashboard_config` 功能不受影响
