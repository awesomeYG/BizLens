## 📋 执行摘要

本方案基于对 **ThoughtSpot、Power BI、Looker、Domo** 四款经典 BI 产品的深度调研，结合当前 BizLens 产品现状，提出差异化产品重新规划。

### 核心结论

**市场机会**: 现有 BI 产品两极分化严重
- **高端市场**: ThoughtSpot/Looker/Domo - 企业级功能复杂，价格昂贵 ($$$$, 需要专业数据团队)
- **低端市场**: Excel/Google Sheets - 灵活但缺乏智能化和协作能力
- **空白市场**: 中小企业的 AI-Native BI - 零学习成本、智能化、用得起

**产品定位**: 
```
不是 "更简单的 BI 工具"
而是 "会数据分析的 AI 业务顾问"
```

**差异化策略**:
1. **业务可观测性** (vs 通用 BI) - 从"查询发生了什么"到"监控业务健康度，预警异常"
2. **AI-First 设计** (vs AI 附加功能) - AI 不是功能，是产品本身
3. **中小企业友好** (vs 企业级) - SaaS 订阅、5 分钟上手、无需数据团队

### 1. ThoughtSpot - AI-Native BI 领导者

#### 核心优势
| 维度 | 说明 | 可借鉴度 |
|------|------|----------|
| **自然语言查询** | "Talk to Your Data" - 行业最先提出 | 🔴 高 |

| **语义层** | 业务指标统一层，让 AI 理解业务含义 | 🔴 最高 |
| **透明 AI** | 显示查询来源、置信度、计算逻辑 | 🟠 中 |
| **嵌入式分析** | SDK 嵌入第三方应用，按用量计费 | 🟢 低 |

#### 技术架构
```
┌─────────────────────────────────────────┐
│           AI Agent Layer                │
│  Spotter | SpotterViz | SpotterModel   │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐

│  业务指标定义、维度层次、计算逻辑         │
```

#### 弱点/机会

- ❌ 价格昂贵 - 不适合中小企业
- ❌ 被动响应 - 用户问才回答，不主动推送

#### 技术特点
```lookml
# LookML 示例 - 指标定义代码化
view: orders {
  measure: gmv {
    type: sum
    sql: ${amount} ;;
  }
  
  measure: order_count {
    type: count
  }
  
  measure: aov {
    type: number
    sql: ${gmv} / ${order_count} ;;
  }
}
```

#### 弱点/机会
- ❌ 需要 SQL/LookML 知识 - 业务人员无法使用

```typescript
  metrics: Metric[];
  dimensions: Dimension[];
  relationships: Relationship[];
}

  id: string;
  name: string;           // "GMV"
  description: string;    // "成交总额"
  formula: string;        // "SUM(orders.amount)"
  dataType: 'currency' | 'number' | 'percentage';
  tags: string[];         // ["销售", "核心指标"]
  autoDetected: boolean;

// 2. AI 查询引擎
interface QueryAgent {

  // 生成可视化
  recommendChart(result: QueryResult): ChartConfig;

// 3. 自动发现指标
async function autoDiscoverMetrics(dataSource: DataSource): Promise<Metric[]> {
  const tables = await dataSource.getTables();
  
  // AI 分析表结构和数据
  const metrics: Metric[] = [];

    });
  }

  return metrics;
```

### 2. 语义层 (核心竞争力) 🔴 最高优先级

#### 为什么重要
- ThoughtSpot/Looker 的核心壁垒
- 让 AI 理解业务含义，不只是表结构
- 支持跨数据源统一指标定义
- 避免"同一个 GMV，不同部门数字不一样"

#### 功能规格

**自动发现**:
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  AI 已发现以下业务指标，请确认：                     │
│                                                    │
```

```typescript
interface AutoLayoutConfig {
  columns: 3;  // 默认 3 列
  rows: 'auto';
  highlight: 'top-left';  // KPI 放左上角
  responsive: true;
}

function autoLayout sections: Section[]): DashboardLayout {
  // 1. KPI 优先，放第一行
  const kpis = sections.filter(s => s.type === 'kpi');
  const layout = placeInRow(kpis, 3);
  
  // 2. 趋势图放第二行 (宽)
  const trends = sections.filter(s => s.type === 'trend');
  layout.push(...placeInRow(trends, 2));
  
  // 3. 排行和地图放第三行
  const others = sections.filter(s => !['kpi', 'trend'].includes(s.type));
  layout.push(...placeInRow(others, 3));
  
  return layout;
}
```

```go
type AlertRule struct {
    ID          string          `json:"id"`
    Name        string          `json:"name"`  // "GMV 低于阈值告警"
    Metric      string          `json:"metric"` // "gmv"
    Condition   AlertCondition  `json:"condition"`
    Schedule    AlertSchedule   `json:"schedule"`
    Notify      NotifyConfig    `json:"notify"`
    Escalation  *EscalationRule `json:"escalation,omitempty"`
    Status      string          `json:"status"`  // active/inactive
}

type AlertCondition struct {
    Operator  string  `json:"operator"`  // <, >, <=, >=, change_percent
    Value     float64 `json:"value"`     // 100000
    CompareTo string  `json:"compareTo"` // "fixed" | "yesterday" | "last_week"
}

type AlertSchedule struct {
    Frequency string `json:"frequency"`  // hourly/daily/weekly
    Time      string `json:"time"`       // "09:00"
    Timezone  string `json:"timezone"`   // "Asia/Shanghai"
}

type NotifyConfig struct {
    Channels []string        `json:"channels"`  // ["feishu", "email", "sms"]
    Receivers []Receiver     `json:"receivers"`
}

type EscalationRule struct {
    TriggerCondition string `json:"trigger"`  // "consecutive_3_times"
    Action           string `json:"action"`   // "notify_ceo"
}
```
