# BizLens 后台管理系统 - 需求文档

> 版本：v1.0.0
> 日期：2026-03-27
> 状态：草稿

## 1. 项目概述

### 1.1 背景与目标

BizLens 后台管理系统（BizLens Admin）是面向企业管理员的运营管理控制台，提供用户管理、数据资产管理、系统配置、操作日志查询等核心功能。该系统作为 BizLens SaaS 平台的核心管理组件，使管理员能够高效完成日常运维工作。

本需求文档在已完成的授权码激活模块（`admin-management` specs）基础上，聚焦于**后台管理的 UI 风格、功能范围、页面结构和交互规范**。

### 1.2 现有基础

已实现的后台管理功能：
- AdminLayout 布局框架（侧边栏 + 顶部导航）
- AdminStats 仪表盘（用户数/文件数/数据源数/存储总量统计）
- 用户管理 CRUD（列表、添加、编辑角色、启用/禁用、删除、重置密码）
- 文件管理（列表、搜索、格式筛选、批量删除）
- 数据库连接管理（列表、搜索、类型筛选、测试连接、删除）
- 存储配置 UI（本地/S3 切换、配置表单、存储统计）

已完成后端 API：
- `/api/admin/stats` - 系统统计
- `/api/admin/users` - 用户管理 CRUD + 重置密码 + 启禁用
- `/api/admin/datasets` - 文件全局列表 + 删除
- `/api/admin/data-sources` - 数据源全局列表 + 测试 + 删除

**待完善项**：
- 系统配置（AI/告警/保留策略）无后端 API 集成
- 存储配置无后端 API 集成
- 缺少操作日志、系统监控、权限管理等高级功能

### 1.3 术语表

| 术语 | 说明 |
|------|------|
| Admin / 管理后台 | BizLens 后台管理系统，指 `/admin/*` 路由下的管理界面 |
| Owner | 所有者角色，系统激活时创建的第一个账号，不可删除 |
| Admin | 管理员角色，可管理用户和数据资产 |
| Member | 成员角色，普通用户，无管理权限 |
| glass-card | 毛玻璃态卡片组件，现有 Admin UI 的核心视觉元素 |

## 2. UI 风格决策

### 2.1 风格决策：**延续现有深色玻璃态风格**

经过对现有前端架构和 UI 风格的全面分析，决策：**后台管理 UI 继承并延续现有的深色玻璃态风格，不做风格切换**。

### 2.2 决策理由

| 维度 | 分析 |
|------|------|
| **产品一致性** | BizLens 整体定位为 AI + 数据分析的智能平台，主产品采用深色专业感风格。Admin 作为产品的一部分，保持风格一致能降低用户的认知成本。 |
| **技术成熟度** | 现有 `glass-card` 组件、`bg-grid` 背景、`indigo` 高亮色、动画类（fade/slide/pulse/shimmer）已在 `globals.css` 中完整定义，可直接复用。 |
| **使用场景** | 中小企业管理员通常在办公室显示器上使用管理后台，深色背景配合高对比度文字更护眼，且在明亮办公室环境中仍有良好可读性。 |
| **视觉精致度** | 现有玻璃态 + 网格 + 光晕背景的组合已具备企业级专业感，无需为了"传统管理后台"的外观而妥协。 |
| **开发效率** | 延续现有风格可最大化复用已有组件和样式定义，减少新造轮子的成本。 |

## 3. 功能范围

### 3.1 功能模块总览

```
后台管理 (Admin)
├── 仪表盘            (/)              [已有，待增强]
├── 用户管理          (/users)         [已有，待完善]
├── 数据资产          (/assets)        [已有，待完善]
│   ├── 文件管理      (/assets/files)
│   ├── 数据库        (/assets/databases)
│   └── 存储配置      (/assets/storage) [已有，待对接API]
├── 系统配置          (/config)        [已有，待对接API]
│   ├── AI 模型配置   (/config/ai)
│   ├── 告警配置      (/config/alerts)
│   ├── 通知渠道      (/config/notifications)
│   └── 数据策略      (/config/retention)
└── 操作日志          (/logs)          [新增]
```

### 3.2 仪表盘增强

**现状**：仅展示用户数/文件数/数据源数/存储总量 4 个统计卡片 + 最近 5 条上传记录。

**增强项**：
- 数据源健康状态概览（正常/异常/未连接的环形图或条形图）
- 近 7 天文件上传趋势（迷你折线图）
- 活跃用户趋势（折线图）
- 系统告警摘要（最近 7 天触发的告警数量 + 环比变化）

### 3.3 用户管理完善

**现状**：表格 + 添加用户弹窗 + 删除确认 + 重置密码显示。

**完善项**：
- 编辑用户弹窗（修改姓名、角色，不允许修改邮箱）
- 用户详情侧滑面板（查看创建时间、最后登录、关联的数据源等）
- 批量启用/禁用
- 按角色筛选（Owner/Admin/Member）
- 用户导入（CSV 批量创建，下载模板）
- 登录锁定状态展示（LockedUntil 字段的剩余时间）

### 3.4 数据资产管理完善

**文件管理**：
- 按租户筛选（显示文件名+所属租户）
- 文件预览（图片/PDF 直接预览，CSV/Excel 预览前 100 行）
- 批量移动到回收站
- 存储统计卡片（按格式分布的饼图或条形图）

**数据库连接**：
- 按租户筛选（显示连接名+所属租户）
- 连接详情弹窗（查看完整连接信息，不显示密码）
- 批量启用/禁用连接
- 定期健康检查（后台任务，显示最后检查时间和结果）

### 3.5 系统配置完善

**现状**：仅有静态表单 UI，无后端 API 对接。

**目标**：与后端 SystemConfig 模型联动，支持增删改查。

| 配置项 | 类型 | 说明 |
|--------|------|------|
| AI 默认模型 | select | gpt-4o-mini / gpt-4o / claude-3-sonnet |
| AI 温度参数 | number | 0.0 ~ 2.0，默认 0.7 |
| 告警冷却时间 | number | 分钟，默认 5 |
| 默认通知渠道 | select | 钉钉/飞书/企业微信/Telegram/无 |
| 文件保留天数 | number | 默认 90 天 |
| 告警日志保留天数 | number | 默认 30 天 |
| 新用户默认角色 | select | admin / member |
| 会话超时时间 | number | 小时，默认 24 |

### 3.6 操作日志（新增）

记录管理员的关键操作行为，支持审计追溯。

| 记录的操作 | 详情 |
|-----------|------|
| 用户管理 | 创建/编辑/删除/启用/禁用/重置密码 |
| 数据资产 | 删除文件/删除数据源/清空回收站 |
| 系统配置 | 修改任何系统配置项 |
| 登录登出 | 管理员账号的登录和登出 |

日志字段：时间戳、操作者、操作类型、目标对象、操作详情（IP 地址）

## 5. 用户角色与权限

### 5.1 角色定义

| 角色 | 说明 | 可访问范围 |
|------|------|----------|
| Owner | 所有者，系统激活时创建，不可删除 | 所有管理功能，不可被删除/降级 |
| Admin | 管理员，由 Owner 或其他 Admin 创建 | 所有管理功能（除修改 Owner） |
| Member | 成员，普通用户 | 无管理后台访问权限 |

### 5.2 权限矩阵

| 功能 | Owner | Admin | Member |
|------|-------|-------|--------|
| 查看仪表盘 | Y | Y | N |
| 查看用户列表 | Y | Y | N |
| 添加用户 | Y | Y | N |
| 编辑用户 | Y | Y | N |
| 删除用户 | Y | Y | N |
| 重置用户密码 | Y | Y | N |
| 启用/禁用用户 | Y | Y | N |
| 管理文件 | Y | Y | N |
| 管理数据库连接 | Y | Y | N |
| 系统配置 | Y | Y | N |
| 查看操作日志 | Y | Y | N |
| 修改系统配置 | Y | Y | N |

### 5.3 前端权限控制

- `AdminLayout` 中通过 `getCurrentUser().role` 判断是否为 admin/owner
- 非 admin/owner 访问 `/admin/*` 时显示"无访问权限"页面
- 后端 `/api/admin/*` 统一使用 JWT 中间件 + 角色校验

## 6. 交互规范

### 6.1 通用交互

| 交互 | 行为 |
|------|------|
| 点击按钮加载中 | 按钮文字改为"加载中..." + `disabled`，操作完成后恢复 |
| 表单保存成功 | Toast 提示"保存成功"（2 秒自动消失） |
| 表单保存失败 | 输入框下方红色错误提示文字 |
| 删除操作 | 先显示模态确认框，标题带红色警示样式 |
| 重置密码 | 模态框显示一次性明文密码，带黄色警示 |
| 批量选择 | 表格左侧 Checkbox，全选/取消全选 |
| 表格排序 | 点击表头切换升/降序，显示方向图标 |

### 6.2 Toast 提示规范

```
位置: 右下角 fixed bottom-6 right-6
样式: glass-card rounded-xl px-4 py-3
动画: slide-up 300ms
颜色: 成功 emerald / 失败 red / 警告 amber / 信息 indigo
时长: 2 秒自动消失
```

### 6.3 空状态规范

```
┌────────────────────────────────┐
│      [图标 64x64 灰色]          │
│      暂无数据                   │
│      text-sm text-zinc-500      │
└────────────────────────────────┘
```

### 6.4 加载状态规范

- 初始加载：骨架屏（`min-h-[40vh] rounded-3xl border border-zinc-900 bg-zinc-950/40`）
- 数据刷新：表格区域顶部 loading 条（indigo 渐变，2px 高）
- 按钮加载：按钮内显示 spinner + "处理中..." 文字

### 6.5 错误处理规范

| 错误类型 | 展示方式 |
|----------|----------|
| 网络错误 | 红色玻璃卡片提示：`加载失败: {message}` + 重试按钮 |
| 权限不足 | 全屏"无访问权限"提示 + 返回首页链接 |
| 操作失败 | Toast 红色提示，1 秒后消失 |
| 表单验证 | 输入框下方红色小字，字段级验证 |
| 404 | 页面级提示"页面不存在" |

## 7. 非功能性需求

### 7.1 性能

- 页面首屏加载（LCP）< 2.5 秒
- API 响应时间 < 500ms（不含复杂查询）
- 表格分页：每页最多 20 条，超过则分页
- 列表页搜索/筛选防抖：300ms

### 7.2 响应式

- `lg` 及以上（>= 1024px）：侧边栏 + 主内容区布局
- `md` (768-1023px)：侧边栏收起为图标模式
- `sm` (< 768px)：侧边栏隐藏，汉堡菜单触发

### 7.3 可访问性

- 所有交互元素有 `:focus-visible` 样式（indigo ring）
- 按钮和链接有 `aria-label`
- 表格有 `scope` 属性
- 支持键盘导航（Tab / Enter / Escape）

### 7.4 兼容性

- 目标浏览器：Chrome 110+、Edge 110+、Firefox 115+、Safari 16+
- Next.js 15 App Router 兼容

## 8. 验收标准

### 8.1 UI 验收

- [ ] 所有 Admin 页面采用统一的 glass-card 风格
- [ ] 侧边栏导航当前路径高亮正确
- [ ] AdminHeaderNav 顶部标签与侧边栏同步
- [ ] 表格、模态框、表单样式与设计系统一致
- [ ] 深色主题在明亮/暗光环境下均正常显示
- [ ] 响应式布局在 lg/md/sm 三档断点均正常

### 8.2 功能验收

- [ ] 仪表盘展示系统核心统计指标
- [ ] 用户管理支持 CRUD + 重置密码 + 启禁用
- [ ] 文件管理支持列表 + 筛选 + 删除 + 批量操作
- [ ] 数据库管理支持列表 + 筛选 + 测试连接 + 删除
- [ ] 系统配置支持 AI/告警/通知/保留策略的配置保存和加载
- [ ] 操作日志记录关键管理操作并支持筛选查询
- [ ] 非 admin/owner 访问 `/admin/*` 显示无权限页面

### 8.3 安全验收

- [ ] 所有 `/api/admin/*` 接口需要 JWT + admin/owner 角色
- [ ] Owner 账号不可被删除或修改角色
- [ ] 重置密码明文仅显示一次
- [ ] API 不返回密码哈希等敏感信息
