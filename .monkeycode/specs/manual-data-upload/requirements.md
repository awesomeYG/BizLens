# 手动上传数据源功能 - 需求文档

**版本**: 1.0  
**日期**: 2026-01-03  
**状态**: 草稿

---

## 1. 文档概述

### 1.1 目的

本文档描述数据分析平台"手动上传数据源"功能的完整需求，支持没有现成数据源的用户通过上传本地文件进行数据分析。

### 1.2 范围

本功能覆盖从文件上传、解析、质检、清洗到数据集管理的完整流程。

### 1.3 目标用户

| 用户类型 | 特征 | 数据现状 |
|---------|------|---------|
| 小微企业 | 没有 IT 系统，用 Excel/纸质记录 | 数据散落在本地文件 |
| 传统行业 | 有业务但数字化程度低 | 数据在员工个人电脑 |
| 初创团队 | 刚起步，还没来得及建系统 | 只有手工整理的表格 |
| 个体商户 | 单打独斗，用简单工具 | 微信聊天记录 + 手写账本 |

## 2. 需求规格

### 2.1 文件上传功能

#### US-001: 单文件上传

**作为** 一个没有数据源的用户  
**我希望** 能够通过拖拽或点击选择的方式上传单个文件  
**以便于** 快速开始数据分析

**验收标准**:
- AC-001.1: 系统应在工作台首页提供明显的"上传数据文件"入口
- AC-001.2: 系统应支持拖拽文件到上传区域触发上传
- AC-001.3: 系统应支持点击上传区域弹出文件选择对话框
- AC-001.4: 系统应在上传过程中显示进度条
- AC-001.5: 系统应在上传完成后显示文件基本信息（名称、大小、行数）

---

#### US-002: 批量文件上传

**作为** 一个需要分析多个月份数据的用户  
**我希望** 能够一次上传多个文件  
**以便于** 减少重复操作，提高效率

**验收标准**:
- AC-002.1: 系统应支持一次选择并上传至少 10 个文件
- AC-002.2: 系统应在上传前询问用户是"合并为一张表"还是"保持独立"
- AC-002.3: 系统应为每个文件显示独立的上传进度
- AC-002.4: 系统应在某个文件上传失败时不影响其他文件的上传
- AC-002.5: 系统应在全部上传完成后显示汇总信息

---

#### US-003: 支持的文件格式

**作为** 一个使用不同工具记录数据的用户  
**我希望** 能够上传多种常见格式的文件  
**以便于** 无需转换格式即可使用

**验收标准**:
- AC-003.1: 系统应支持 Excel 格式（.xlsx, .xls）
- AC-003.2: 系统应支持 CSV 格式（.csv）
- AC-003.3: 系统应支持 JSON 格式（.json）
- AC-003.4: 系统应支持 XML 格式（.xml）
- AC-003.5: 系统应支持 PDF 表格（可解析的.pdf）
- AC-003.6: 系统应支持 Word 表格（.docx）
- AC-003.7: 系统应在用户上传不支持的格式时给出友好提示

---

### 2.2 数据模板功能

#### US-004: 模板下载

**作为** 一个不熟悉数据规范的用户  
**我希望** 能够下载预置的数据模板  
**以便于** 按照正确格式整理数据

**验收标准**:
- AC-004.1: 系统应提供"销售数据模板"下载
- AC-004.2: 系统应提供"客户信息模板"下载
- AC-004.3: 系统应提供"库存数据模板"下载
- AC-004.4: 系统应提供"财务数据模板"下载
- AC-004.5: 系统应在模板中包含字段说明和填写示例
- AC-004.6: 系统应在上传页面提供模板下载入口

---

### 2.3 数据解析功能

#### US-005: 智能格式识别

**作为** 一个上传文件的用户  
**我希望** 系统能够自动识别文件格式  
**以便于** 无需手动指定文件类型

**验收标准**:
- AC-005.1: 系统应根据文件扩展名和文件内容自动识别格式
- AC-005.2: 系统应能识别 Excel 文件中的多个工作表
- AC-005.3: 系统应能识别 CSV 文件的编码（UTF-8, GBK 等）
- AC-005.4: 系统应能识别 CSV 文件的分隔符（逗号、制表符等）
- AC-005.5: 系统应在识别失败时允许用户手动选择格式

---

#### US-006: 字段类型推断

**作为** 一个上传数据的用户  
**我希望** 系统能够自动推断每个字段的类型  
**以便于** 无需手动配置字段属性

**验收标准**:
- AC-006.1: 系统应能推断字符串类型（String）
- AC-006.2: 系统应能推断整数类型（Integer）
- AC-006.3: 系统应能推断浮点数类型（Float）
- AC-006.4: 系统应能推断日期类型（Date）
- AC-006.5: 系统应能推断布尔类型（Boolean）
- AC-006.6: 系统应能识别主键字段（唯一值字段）
- AC-006.7: 系统应在推断结果页面展示字段类型供用户确认

---

### 2.4 数据质检功能

#### US-007: 数据质量检查

**作为** 一个上传数据的用户  
**我希望** 系统能够自动检查数据质量问题  
**以便于** 在分析前发现并修复问题

**验收标准**:
- AC-007.1: 系统应检测空值并显示空值比例
- AC-007.2: 系统应检测格式不一致（如日期格式混用）
- AC-007.3: 系统应检测异常值（超出合理范围的数值）
- AC-007.4: 系统应检测重复数据
- AC-007.5: 系统应检测字段命名问题（空格、特殊字符）
- AC-007.6: 系统应以列表形式展示所有发现的问题
- AC-007.7: 系统应为每个问题提供严重程度标记（高/中/低）

---

### 2.5 数据清洗功能

#### US-008: 数据清洗向导

**作为** 一个发现数据质量问题的用户  
**我希望** 系统能够提供一键修复或引导修复  
**以便于** 轻松完成数据清洗

**验收标准**:
- AC-008.1: 系统应提供"统一日期格式"的修复选项
- AC-008.2: 系统应提供"提取数值"的修复选项（如从"¥1,000"提取 1000）
- AC-008.3: 系统应提供"删除重复数据"的修复选项
- AC-008.4: 系统应提供"填充空值"的修复选项（平均值/中位数/指定值）
- AC-008.5: 系统应在修复前预览变更内容
- AC-008.6: 系统应支持撤销清洗操作

---

### 2.6 数据集管理功能

#### US-009: 数据集列表

**作为** 一个已上传数据的用户  
**我希望** 能够查看我上传的所有数据集  
**以便于** 管理和选择要分析的数据

**验收标准**:
- AC-009.1: 系统应列出用户上传的所有数据集
- AC-009.2: 系统应区分"本地上传"和"已连接数据源"
- AC-009.3: 系统应显示每个数据集的基本信息（名称、上传日期、大小、行数）
- AC-009.4: 系统应提供搜索功能
- AC-009.5: 系统应提供按上传日期排序功能

---

#### US-010: 数据集操作

**作为** 一个已上传数据的用户  
**我希望** 能够对数据集进行管理操作  
**以便于** 保持数据的时效性和准确性

**验收标准**:
- AC-010.1: 系统应支持预览数据集内容（前 100 行）
- AC-010.2: 系统应支持更新数据集（上传新文件替换）
- AC-010.3: 系统应支持删除数据集
- AC-010.4: 系统应在删除前要求用户确认
- AC-010.5: 系统应显示数据集被哪些分析使用
- AC-010.6: 系统应在数据集被使用时阻止删除或级联删除

---

#### US-011: 数据集版本管理

**作为** 一个频繁更新数据的用户  
**我希望** 系统能够保留历史版本  
**以便于** 在需要时回退到之前的版本

**验收标准**:
- AC-011.1: 系统应在每次更新时保留历史版本
- AC-011.2: 系统应保留最近 5 个版本
- AC-011.3: 系统应显示版本历史（版本号、更新时间、变更摘要）
- AC-011.4: 系统应支持回退到任意历史版本
- AC-011.5: 系统应支持删除历史版本

---

### 2.7 安全与隐私功能

#### US-012: 数据传输加密

**作为** 一个关心数据安全的用户  
**我希望** 上传的数据在传输过程中加密  
**以便于** 防止数据被窃取

**验收标准**:
- AC-012.1: 系统应使用 HTTPS/TLS 加密传输
- AC-012.2: 系统应支持 TLS 1.2 及以上版本
- AC-012.3: 系统应在不安全连接时阻止上传

---

#### US-013: 数据存储加密

**作为** 一个关心数据安全的用户  
**我希望** 上传的数据在存储时加密  
**以便于** 防止数据泄露

**验收标准**:
- AC-013.1: 系统应使用 AES-256 加密存储
- AC-013.2: 系统应加密存储文件内容
- AC-013.3: 系统应加密存储文件元数据

---

#### US-014: 访问控制

**作为** 一个上传数据的用户  
**我希望** 只有我能访问我上传的数据  
**以便于** 保护我的商业机密

**验收标准**:
- AC-014.1: 系统应确保只有上传者可访问该数据集
- AC-014.2: 系统应支持数据共享（可选功能）
- AC-014.3: 系统应记录数据访问日志
- AC-014.4: 系统应提供访问日志查询功能

---

#### US-015: 敏感数据处理

**作为** 一个上传包含敏感信息数据的用户  
**我希望** 系统能够识别并脱敏敏感数据  
**以便于** 符合隐私保护要求

**验收标准**:
- AC-015.1: 系统应能识别身份证号
- AC-015.2: 系统应能识别手机号
- AC-015.3: 系统应能识别银行卡号
- AC-015.4: 系统应提供脱敏选项（部分隐藏/完全隐藏）
- AC-015.5: 系统应在数据预览时默认脱敏显示

---

### 2.8 OCR 功能（可选高级功能）

#### US-016: 截图/扫描件识别

**作为** 一个只有纸质记录或截图的用户  
**我希望** 系统能够通过 OCR 识别图片中的表格  
**以便于** 将纸质数据数字化

**验收标准**:
- AC-016.1: 系统应支持 JPG、PNG 格式图片上传
- AC-016.2: 系统应能识别图片中的表格结构
- AC-016.3: 系统应能识别简体中文文字
- AC-016.4: 系统应能识别数字和日期
- AC-016.5: 系统应显示 OCR 识别结果供用户校对
- AC-016.6: 系统应标注识别置信度
- AC-016.7: 系统应支持手动修正识别错误

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 单文件上传大小限制 | ≤ 100MB |
| 批量上传文件数量 | ≤ 10 个/次 |
| 上传响应时间 | < 3 秒（10MB 文件） |
| 解析响应时间 | < 10 秒（10000 行数据） |
| 质检响应时间 | < 5 秒（10000 行数据） |
| 页面加载时间 | < 2 秒 |

### 3.2 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥ 99.9% |
| 上传成功率 | ≥ 99% |
| 解析准确率 | ≥ 95% |
| OCR 识别准确率 | ≥ 90%（清晰图片） |

### 3.3 安全需求

| 要求 | 说明 |
|------|------|
| 传输加密 | HTTPS/TLS 1.2+ |
| 存储加密 | AES-256 |
| 访问控制 | 基于用户身份认证 |
| 日志记录 | 记录所有数据访问操作 |
| 数据隔离 | 不同用户数据物理或逻辑隔离 |

### 3.4 兼容性需求

| 类别 | 要求 |
|------|------|
| 浏览器 | Chrome 80+, Firefox 75+, Safari 13+, Edge 80+ |
| Excel 版本 | Excel 2007 及以上 |
| 移动端 | 支持主流移动浏览器（仅预览功能） |

## 4. 用户界面需求

### 4.1 上传页面

- 明显的上传入口（按钮或拖拽区域）
- 支持的文件格式提示
- 上传进度显示
- 模板下载入口

### 4.2 数据预览页面

- 表格形式展示前 100 行数据
- 字段类型显示
- 统计信息（行数、列数、空值比例等）
- 数据质量问题列表

### 4.3 清洗向导页面

- 问题列表及严重程度
- 修复建议及预览
- 一键修复按钮
- 撤销操作按钮

### 4.4 数据集管理页面

- 数据集列表（卡片或表格）
- 搜索和筛选功能
- 操作按钮（预览、更新、删除、版本历史）
- 使用情况提示
