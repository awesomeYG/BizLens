## 📋 测试步骤

### 1. 访问注册页面
打开浏览器访问：http://localhost:3000/auth/register

填写注册表单：
- 租户 ID: demo
- 姓名：测试用户
- 邮箱：test@example.com
- 密码：password123
- 确认密码：password123

### 2. 访问登录页面
打开浏览器访问：http://localhost:3000/auth/login

填写登录表单：
- 邮箱：test@example.com
- 密码：password123

### 3. 测试受保护路由
登录成功后，访问任何受保护的页面会自动验证 Token

### 4. API 测试

使用 curl 测试后端 API：

```bash
# 注册
curl -X POST http://localhost:3001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"tenantId":"demo","name":"测试","email":"test@example.com","password":"password123"}'

# 登录
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'

# 获取当前用户（需要 Token）
curl -X GET http://localhost:3001/api/auth/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## ⚠️ 注意事项

### 数据库配置
当前使用 SQLite 内存数据库，重启后数据会清空。

**生产环境建议使用：**
```bash
# PostgreSQL
docker run -d --name postgres \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=ai_bi \
  -p 5432:5432 \
  postgres:15
```

### Token 配置
- Access Token 有效期：30 分钟
- Refresh Token 有效期：7 天
- 可在 .env 中修改

### 安全特性
- ✅ 密码 BCrypt 加密
- ✅ 连续 5 次登录失败锁定账户 30 分钟
- ✅ JWT Token 签名验证
- ✅ CORS 跨域配置
