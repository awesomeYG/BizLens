# BizLens 测试数据使用指南

## 🚀 快速开始（推荐）

### 方式 1: 一键快速测试（最简单）

1. 启动应用
```bash
./start.sh
```

2. 访问首页 http://localhost:3000

3. 点击 **"🚀 快速测试（使用 Mock 数据）"** 按钮

4. ✅ 自动登录并进入主界面，已预填充：
   - 企业信息：杭州智选电商有限公司
   - 企业画像：完整的分析建议
   - 数据源：MySQL 生产库 + CSV 营销数据

### 方式 2: 使用调试工具

1. 访问调试页面 http://localhost:3000/debug

2. 点击 **"快速登录（Mock 数据）"** 按钮

3. 自动跳转到首页

### 方式 3: 运行测试脚本

```bash
# 查看测试数据说明
./scripts/seed-test-data.sh

# 按提示操作
```

## 📦 Mock 数据内容

### 企业信息
```json
{
  "companyName": "杭州智选电商有限公司",
  "industry": "电商零售",
  "size": "200-500 人",
  "region": "华东地区 + 东南亚",
  "businessModel": "B2C D2C 混合模式",
  "coreGoals": "提升复购率至 35%，优化库存周转至 15 天，降低获客成本 20%"
}
```

### 企业画像
- **总结**: 专注于家居生活品类的电商企业，年 GMV 约 3 亿元
- **分析焦点**:
  - 销售增长趋势与渠道贡献分析
  - 库存周转效率与滞销预警
  - 客户复购率与生命周期价值
- **推荐指标**:
  - GMV、毛利率、客单价、ROI
  - 库存周转天数、动销率、滞销占比
  - 新客占比、复购率、客诉率

### 数据源
1. **生产数据库 - MySQL**
   - 类型：MySQL
   - 表：orders, order_items, products, categories, customers, inventory, shipments
   - 状态：已连接

2. **营销投放数据 - CSV**
   - 类型：CSV
   - 描述：从巨量引擎导出的广告投放数据
   - 状态：已连接

## 🛠️ 调试工具功能

访问 `/debug` 页面可以使用以下功能：

### 1. 快速登录
- 使用 Mock 数据一键登录
- 跳过 onboarding 流程
- 直接进入主界面

### 2. 清除所有数据
- 删除 localStorage 中的所有数据
- 恢复到初始状态
- 用于测试新用户流程

### 3. 重置 Onboarding
- 保持登录状态
- 重新显示初始化向导
- 用于测试 onboarding 流程

### 4. Mock 模式开关
- 启用/禁用 Mock 模式
- 用于开发测试

## 📝 测试场景

### 场景 1: 测试 AI 对话功能
1. 点击首页"快速测试"按钮
2. 进入 `/chat` 页面
3. 上传测试数据或选择已连接的数据源
4. 提问："帮我分析销售趋势"

### 场景 2: 测试数据源管理
1. 点击首页"快速测试"按钮
2. 进入 `/data-sources` 页面
3. 查看预填充的数据源
4. 点击"测试连接"验证

### 场景 3: 测试完整 onboarding 流程
1. 访问 `/debug` 页面
2. 点击"清除所有数据"
3. 刷新首页
4. 手动输入姓名邮箱登录
5. 完成企业信息填写
6. 选择数据源类型
7. 生成企业画像

### 场景 4: 测试数据大屏
1. 点击首页"快速测试"按钮
2. 进入 `/dashboards` 页面
3. 选择模板生成大屏
4. 切换不同 Tab 查看

## 🔧 开发建议

- 预填充的数据更有说服力
